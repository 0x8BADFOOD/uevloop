Detailed Description

Events are special closures passed around the core. They represent tasks to be run at some point by the system.

Events are bound to information on how and when they should be invoked. They can be timers, which are scheduled to be run in the future, or signals, which represent asynchronous reactions to occurances thorough the system.

Both timers and signals can be recurring, in which case they won't be discarded after processing by the event loop.

Events can also be lifeless shells to closures, in which case they will be immediately destroyed after execution.
